DataQualityDashboard Tutorial on a Staged OMOP CDM

Introduction

This tutorial walks through a staged DQD workflow on a synthetic OMOP CDM built from Synthea. The tutorial data has six curated issues distributed across fatal, convention, and characterization severities.

Each stage follows the same pattern:

  1. run DQD for the current severity
  2. inspect the failures
  3. query the CDM to find the bad rows
  4. apply the fix
  5. rerun DQD to verify the fix

The prepared snapshots keep each stage deterministic without mutating the clean baseline.

Tutorial Assets and Snapshot Overview

The tutorial uses these prepared databases:

  • data/syntheaCDM_tutorial_broken.duckdb: contains all injected issues
  • data/syntheaCDM_tutorial_after_fatal.duckdb: fatal issues already fixed
  • data/syntheaCDM_tutorial_after_convention.duckdb: fatal and convention issues already fixed
Note

Each stage works on a temporary copy of the corresponding snapshot. That lets you rerun chunks without corrupting the prepared reference databases.

Overlapping observation periods

Code 
cat("```sql\n", get_issue_sql("fatal_observation_period_overlap"), "\n```", sep = "")

      SELECT
        op1.person_id,
        op1.observation_period_id AS first_observation_period_id,
        op2.observation_period_id AS second_observation_period_id,
        op1.observation_period_start_date AS first_start_date,
        op1.observation_period_end_date AS first_end_date,
        op2.observation_period_start_date AS second_start_date,
        op2.observation_period_end_date AS second_end_date
      FROM main.observation_period op1
      INNER JOIN main.observation_period op2
        ON op1.person_id = op2.person_id
       AND op1.observation_period_id < op2.observation_period_id
       AND op1.observation_period_start_date <= op2.observation_period_end_date
       AND op2.observation_period_start_date <= op1.observation_period_end_date
      ORDER BY op1.person_id

Observations outside the visit window

Code 
cat("```sql\n", get_issue_sql("characterization_observation_within_visit_dates"), "\n```", sep = "")

      SELECT
        o.observation_id,
        o.visit_occurrence_id,
        o.observation_date,
        v.visit_start_date,
        v.visit_end_date
      FROM main.observation o
      INNER JOIN main.visit_occurrence v
        ON o.visit_occurrence_id = v.visit_occurrence_id
      WHERE o.observation_date < (v.visit_start_date - 7)
         OR o.observation_date > (v.visit_end_date + 7)
      ORDER BY o.observation_id
